# 2026-02-03 — Portal Tema/WYSIWYG Çalışmaları (Kalıcı Not)

Bu dosya, bugün yapılan tema + WYSIWYG (editör içinden gerçek canlı ekranı görerek düzenleme) çalışmalarını ve kalıcı ayar alanlarını tek yerde toplar.

## Bu ayarlar için “komple yeniden başlatma” gerekir mi?

Hayır. 2 ayrı konu var:

1) **Ayarın DB’ye kaydolması**: Editör `settings.theme` altına yazıyorsa, bu kalıcıdır. Event’i yeniden oluşturmak veya “komple sıfırdan başlamak” gerekmez.

2) **Kodun bu ayarı uygulaması**: Eğer canlı ekran kodu yeni alanları (logo/toggle/font/qrPos) daha yeni öğrendiyse, bu değişikliklerin görünmesi için yalnızca **portal’ın deploy/rebuild edilmesi** gerekir.

Bugün yapılan değişiklikler için üretim ortamında pratik çözüm:
- Sadece portal’ı rebuild edip restart etmek yeterli.
- Bu workspace’te servis adı: `soruyorum-portal` (compose dosyası: `site/docker-compose.yml`).

Kullandığımız komut (prod):
- `cd /srv/webhosting/soruyorum.online/site`
- `docker compose -f docker-compose.yml -f docker-compose.traefik.yml up -d --build soruyorum-portal`

Not: Bu komut Compose bake nedeniyle `soruyorum-api-server` image’ını da build edebiliyor; ama ana hedef portal’ın yeni kodla ayağa kalkmasıdır.

Tarayıcı tarafı:
- Live sayfa açıkken **hard refresh** (Ctrl+Shift+R) gerekebilir.
- Editördeki iframe preview bazen cache’te kalmış HTML/JS ile kalabilir; iframe’i yenilemek çözebilir.

## Hedef
- Editörde (Dashboard /events/[id]/edit) tema ayarlarını değiştirince **canlı ekranın aynısını** (WYSIWYG) gör.
- Seçimler yalnızca local state’te kalmasın; event settings’e yazılsın ve canlı ekran onları kullansın.

## Karar / Standart
- Cloudflare Images URL’leri `imagedelivery.net/.../<imageId>/soruyorum` variant ile standardize.
  - Variant hedefi: **1920×1080** (sunum ekranı).

## Kalıcı Theme Alanları
Tema ayarları event üzerinde `settings.theme` altında tutulur (uygulama bazı yerlerde `event.theme` veya `event.settings.theme` okuyabiliyor; live sayfada fallback var).

- Görsel tema:
  - `backgroundImage` (URL)
  - `background` (legacy/alternatif)
- Düz renk tema:
  - `backgroundColor`
- Renkler:
  - `primaryColor`
  - `colorPalette` (light/dark vb)
  - `textColor` (son düzeltme: canlı ekran bunu öncelikli kullanır)
- Tipografi:
  - `fontFamily`
- Logolar:
  - `logoUrl` (legacy)
  - `logo`: `{ url, x, y, size }`
  - `rightLogo`: `{ url, x, y, size, anchor }`
- Sunum (toggle) ayarları:
  - `showInstructions`
  - `showQR`
  - `showStats`
  - `showNames`
  - `bgAnimation`
- QR konumu:
  - `qrPos`: `{ x, y }` (px)

Not: Bu alanların bir kısmı editörden kaydediliyor; canlı ekran tarafında da okunup uygulanır.

### Örnek `settings.theme` (temsilî)

```json
{
  "backgroundImage": "https://imagedelivery.net/.../abc123/soruyorum",
  "primaryColor": "#7c3aed",
  "colorPalette": "koyu",
  "textColor": "#ffffff",
  "fontFamily": "Poppins",
  "showInstructions": true,
  "showQR": true,
  "showStats": true,
  "showNames": false,
  "bgAnimation": true,
  "qrPos": { "x": 40, "y": 40 },
  "logo": { "url": "data:image/png;base64,...", "x": 0, "y": 0, "size": 160 },
  "rightLogo": { "url": "data:image/png;base64,...", "x": 0, "y": 0, "size": 160, "anchor": "top-right" }
}
```

## WYSIWYG / Preview
- Live sayfa: `apps/portal/src/app/(presentation)/events/[id]/live/page.tsx`
  - `?view=join|wall|rotate` ile görünümü zorlayabilir.
  - `?embed=1` ile iframe embed modunda scroll/margin kapatma.
  - Soru yoksa forced wall/rotate preview için demo sorular.
  - `settings.theme.textColor` varsa onu text rengi olarak kullan.

- Edit sayfa: `apps/portal/src/app/(dashboard)/events/[id]/edit/page.tsx`
  - Canlı ekran iframe ile ortada WYSIWYG önizleme.
  - Iframe “cuk otursun” diye sabit 1920×1080 viewport + container’a göre otomatik scale (ScaledIframe).
  - Tema sekmesi “Kahoot benzeri”: kategori + grid + tema önizleme slaytları (tıkla-uygula).
  - Sunum ayarları / logo / font seçimleri `settings.theme` içine yazılır.

## Bugün yapılan teknik değişiklikler (detaylı)

### 1) Live ekran yeni theme alanlarını uyguluyor

Dosya: `site/apps/portal/src/app/(presentation)/events/[id]/live/page.tsx`

Eklenen/bağlanan davranışlar:
- `showInstructions`: üst talimat barını aç/kapat.
- `showQR`: QR/PIN kutusunu aç/kapat.
- `showStats`: katılımcı sayısı gibi widget’ları aç/kapat.
- `showNames`: katılımcı isimlerini aç/kapat.
- `bgAnimation`: arka plan blur/pulse overlay.
- `qrPos`: QR kutusunun konumunu (px) ayarla.
- `fontFamily`: root container’a font-family uygula (basit map + serbest string desteği).
- `logo` / `rightLogo` + fallback `logoUrl`: logoları göster/konumlandır.

### 2) Editör tarafı (önceden yapılan) persist/hydrate yaklaşımı

Dosya: `site/apps/portal/src/app/(dashboard)/events/[id]/edit/page.tsx`

Editörde şunların `settings.theme` içine yazılması hedeflendi/bağlandı:
- `textColor` (özellikle beyaz seçimi)
- `fontFamily`
- `logo`, `rightLogo`
- `qrPos`
- `showInstructions/showQR/showStats/showNames/bgAnimation`

Kritik prensip:
- Tema uygularken `settings.theme` komple ezilmesin; **merge** yapılarak `textColor` gibi ek alanlar kaybolmasın.

## Doğrulama checklist (prod)

1) Edit ekranında bir event aç: `/events/[id]/edit`
2) Tema sekmesinde:
  - Yazı rengi değiştir → Kaydet → live/iframe’de yazı rengi değişmeli.
  - Font seç → Kaydet → live’da tipografi değişmeli.
  - Logo yükle → Kaydet → live’da sol/sağ logo görünmeli.
3) Sunum Ayarları:
  - Talimat barını kapat → live’da üst bar kaybolmalı.
  - QR kapat → QR kutusu kaybolmalı.
  - Katılımcı bilgisi kapat → sayaç kaybolmalı.
  - İsimleri kapat → isim alanı görünmemeli.
  - Arka plan animasyonu aç/kapat → blur/pulse overlay değişmeli.
4) Eğer değişiklik görünmüyorsa:
  - Live sayfayı hard refresh et.
  - Portal servisini rebuild et (`soruyorum-portal`).
